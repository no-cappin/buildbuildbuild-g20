Here's an overview of our project.

The Build, Build, Build campaign was an initiative started by the previous Duterte administration, "aimed at achieving a golden age of infrastructure in the Philippines to enhance mobility and connectivity, as well as spurring economic growth" (Patinio, 2022). However, there have been several projects that have since been misattributed to this campaign. One such project is the creation of the Cebu-Cordova Link Expressway (CCLEX), which started development under previous president Benigno Aquino III. The project aims to discover patterns on tweets that incorrectly credit this infrastructure to the Build, Build, Build campaign, with a greater objective of combating the rampant mis/disinformation plague in social media today.

Research Question

Which Twitter account type garnered the most engagements on tweets about the misattribution of the Cebu-Cordova Link Expressway (CCLEX) to the Build, Build, Build campaign of the Duterte administration?

Hypothesis

Anonymous accounts obtained significantly higher engagements from tweets that misattributed CCLEX to the Build, Build, Build campaign.

Null Hypothesis

There is no significant difference among the engagements on misattributing tweets about CCLEX, among anonymous Twitter accounts, and other Twitter account types.

Action Plan

Analyze Twitter account types that published mis/disinformation tweets about the Build, Build, Build campaign.

Let's talk about our data science methodology.

Data Preprocessing

Prior to data exploration, several preprocessing steps were necessary to prepare the data.

Ensuring Formatting Consistency
For this step, the content type of the tweets were verified. Particularly, tweets that were both rational and emotional were altered to have the same format of “Rational, Emotional” to ensure that the code will classify the tweets into one category.

Normalization/Standardization
Standardization was done with the tweets’ likes, retweets, replies, and quote tweets, by getting the z-score of the tweets in terms of these four features. This was also done to identify possible outliers in the dataset.

Handling Outliers
For this study, outliers were considered as tweets that attained a z-score of 3.00 and above, in terms of likes, replies, retweets, and quote tweets. Upon standardization, 3 outliers were discovered. Since this is merely a minimal percentage of the tweets gathered, the researchers decided to omit these tweets, bringing the total number of tweets in the dataset to 152.

Categorical Data Encoding
Additionally, certain categorical features in the dataset were converted to a quantifiable value. This is particularly seen with the tweet types feature. A value of 1 was set for tweets who included a certain media type (e.g. photo/video, URL), while a value of 0 indicates the absence of such.

Handling Missing Values
Upon reviewing the data collected, three features were found to be inconsistent.
Views. The view count of most tweets were unavailable during data collection; thus, the researchers decided to disregard this tweet feature for the subsequent parts of the study.
Location. The account locations of the tweet authors were inconsistent. Some accounts had dubious locations, while others did not provide a location in their account. Meanwhile, most accounts had the location “Philippines” or “Republic of the Philippines,” which are broad, and thus, will not provide much insight with the dataset. Thus, the researchers have also decided to dismiss this tweet feature.
Account Bio. The account bios of tweet authors were also inconsistent. Similar to account location, some accounts did not have a bio. Meanwhile, other account bios do not provide significant insight into the study, as they were unrelated to the topic. Thus, the researchers have decided to omit this tweet feature.

Natural Language Processing

The Natural Language Toolkit (NLTK) library was used. First, the text was converted to lowercase and punctuations were removed using built-in string methods. The text was then translated to English using google translate and the emojis and emoticons were converted to plain text. Built-in functions from NLTK were utilized for tokenization and the stopwords were removed. Finally, the data was stemmed and lemmatized by also using the NLTK library.
